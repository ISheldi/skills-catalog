# University Template — Zettelkasten-база знаний

Переносимый шаблон базы знаний, управляемой по строгим правилам (`METHODOLOGY.md`). Работает в связке с Claude Code: кидаешь ссылку → Claude обрабатывает источник в 10 шагов → результат раскладывается по структурированным папкам с перекрёстными ссылками.

Это **не скрипты**, а конвенция + шаблоны: сам пайплайн выполняет Claude, прочитав методологию.

## Артефакты пайплайна

- `sources/<id>/` — первоисточники с конспектами
- `notes/n-<slug>.md` — атомарные заметки (одна идея = один файл)
- `claims/c-<NNNN>.md` — проверяемые утверждения с доказательствами
- `syntheses/syn-<topic>.md` — сводные документы
- `contradictions/contr-<NNN>.md` — зафиксированные конфликты
- `verdicts/v-<slug>.md` — итоговые решения
- `experiments/` — собственные бэктесты и симуляции
- `graph/edges.jsonl` — машиночитаемый граф связей

## Установка

```bash
git clone https://github.com/yasikvlad/university-template.git ~/university-template
```

Затем в любом проекте запускаешь `~/university-template/bootstrap.sh <project-path>` — он разворачивает скелет базы знаний.

## Зависимости

- `bash` 4+, `python3` 3.6+ (только stdlib), GNU coreutils
- Claude Code — шаблон работает и без него, но вся ценность в связке с Claude

## Когда использовать

Когда в проекте нужно системно собирать, проверять и связывать знания из множества источников — вместо хаотичных заметок строить проверяемую базу с доказательствами и графом связей.
