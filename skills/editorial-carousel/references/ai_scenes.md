# AI-сцены — генерация фоновых картинок (режимы D / E)

Когда у человека нет своих фото, но он хочет картинку на обложку/слайды — генерируем её. Текст всё равно рисует браузер поверх (со скримом), поэтому **картинка = фон**, а не финальный слайд.

## Когда какой режим
В конфигуляторе у AI-слайда есть подвыбор `aiMode`: **`auto`** → режим D, **`ref`** → режим E.

- **D — ИИ сам (по идее, `aiMode:"auto"`).** Реферанса нет. Мы предлагаем **3 разных варианта сцены и стиля** под слайд/тему → человек выбирает один → генерим.
- **E — по референсу (`aiMode:"ref"`).** Человек загрузил картинку-реф (`file`). Снимаем с неё **стиль + цвет + настроение**, предлагаем 1–2 идеи сцены под карусель → генерим **в этом стиле** (реф уходит в генерацию как `--image`).
  - Если нужен только бренд-цвет (без сгенерированного фото) — это **режим F**: извлекаем палитру и собираем кастомную тему (см. `themes.md`), без генерации.

## Правила промпта (важно для читаемости текста)
- **Вертикаль 4:5**, субъект в верхней/центральной части — низ слайда уйдёт под скрим + текст.
- **Оставляй «воздух»** (негативное пространство) под заголовок.
- **Без текста на картинке** — текст добавляет браузер.
- Передавай **палитру темы** (hex) и настроение, чтобы фон жил с темой.
- Светлый текст на тёмном скриме → выбирай сцены, где низ не слишком пёстрый.

## Два пути генерации (человек выбирает)

### Путь 1 — «сам генерю» (ChatGPT / любой сервис), бесплатно для наших инструментов
1. Мы выдаём **готовые промпты** (для D — 3 штуки, для E — 1–2 в стиле рефа).
2. Человек генерит у себя, сохраняет файл в папку (напр. `raw/cover_ai.jpg`).
3. Дальше как с обычным фото: `python scripts/prep_media.py photo raw/cover_ai.jpg raw/cover_crop.jpg --anchor top`, затем в спеке `"background": {"kind":"photo","src":"raw/cover_crop.jpg"}`.

### Путь 2 — Higgsfield CLI (авто), платно (кредиты)
> Перед запуском **подтверди с человеком** — генерация тратит кредиты. Сначала прочитать официальный `higgsfield-generate/SKILL.md` (правило памяти).

**Bootstrap:** убедиться, что `higgsfield` в PATH и есть авторизация (`higgsfield account status`; если нет — попросить `higgsfield auth login`).

**D — по идее** (дефолтная модель для дизайна/общей генерации — GPT Image 2):
```bash
higgsfield generate create gpt_image_2 \
  --prompt "<выбранная сцена; вертикаль; палитра #..; без текста; воздух сверху для заголовка>" \
  --aspect_ratio 3:4 --resolution 2k --wait
```
(1080×1350 = 4:5; ближайший допустимый у Higgsfield — `3:4`. Потом всё равно кропим под размер.)

**E — по референсу** (reference-driven — Nano Banana 2; реф уходит флагом `--image`):
```bash
higgsfield generate create nano_banana_2 \
  --prompt "<сцена в стиле рефа; та же палитра/настроение; вертикаль; без текста>" \
  --image raw/ref.jpg --aspect_ratio 3:4 --wait
```

**После генерации (оба пути):** скачать картинку по URL в `raw/`, затем тот же шаг, что и для фото:
```bash
python scripts/prep_media.py photo raw/<downloaded>.jpg raw/cover_crop.jpg --anchor top
```
и в спеке слайда — `"background": {"kind":"photo","src":"raw/cover_crop.jpg"}`.

## Цикл подтверждения (обязательно)
После **каждой** генерации — покажи результат человеку и спроси: **«подходит или перегенерировать?»**
- Не подошло → уточни, что не так (композиция/цвет/настроение), поправь промпт и сгенерь снова. Повторяй, пока не ок.
- Только после «ок» кропим (`prep_media.py photo`) и ставим в спеку.
Никогда не вставляй сгенерированную картинку в карусель без явного «подходит».

## Сколько слайдов генерить
Обычно **обложка** (+ 1–2 ключевых слайда). Не делай все слайды генеренными — микс сгенеренного фона со статикой выглядит дороже (та же логика, что для видео-слайдов).
