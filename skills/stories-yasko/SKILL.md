# Скилл: Создание линий сторис по методу Влада Яско

## Когда использовать

Этот скилл активируется при любом запросе, связанном с:
- Созданием линий Instagram-сторис (экспертные, продающие, обучающие, прогревающие)
- Написанием сценариев прогревающих/продающих/обучающих сторис
- Планированием сторис на день или неделю
- Созданием «умных линий одного дня»
- Переупаковкой контента (видео, статьи, мастер-классы) в формат сторис
- Прогревом к запуску продукта через сторис

## Философия

Ты проектируешь **путешествие зрителя** от первого кадра до призыва к действию. Не полотно текста, а компактную, цепляющую серию.

Четыре главных принципа:
1. **Структура > Импровизация** — каждая линия имеет архитектуру: вступление → основной блок → заключение
2. **Образ > Информация** — люди покупают не знания, а желание быть похожим на эксперта
3. **Краткость > Полотно** — искусство сторис = вовлечь и удержать МАЛЫМ количеством. Каждая лишняя сторис снижает досматриваемость
4. **Разговор > Текст** — сторис = это прежде всего РАЗГОВОР с камерой. Большая часть контента передаётся голосом (говорящая голова), а не текстом на слайдах. Текстовые слайды — это заголовки, акценты и паузы между разговорными блоками. Зритель хочет видеть живого человека, а не читать текст

## Процесс работы

### Фаза 1: Сбор входных данных

Получи от пользователя:
1. **Тема линии сторис** (о чём будет серия?)
2. **Целевая аудитория** (кто будет смотреть?)
3. **Тип линии** (обучающий прогрев / продажа / сюжетная линия / умная мысль / инфоповод)
4. **Цель** (прогреть к покупке / дать ценность / повысить охваты / вовлечь)
5. **Контекст эксперта** (ниша, продукт, результаты, кейсы учеников)
6. **Есть ли продукт для продажи?** (если да — название, цена, что даёт)

Если пользователь не указал что-то — спроси. Без контекста эксперта сторис будут шаблонными.

### Фаза 2: Выбор типа и структуры

На основе входных данных определи тип линии (см. `references/structure.md`):

| Тип | Когда использовать | Длина |
|---|---|---|
| Умная линия одного дня | Экспертный разбор темы с обучением | 15-25 сторис |
| Линия одного дня (сюжетная) | Событие, обзор, встреча | 10-18 сторис |
| Умная мысль | Один инсайт/лайфхак | 7-12 сторис |
| Инфоповод | Яркое событие + смыслы | 10-18 сторис |
| Продающая линия | Прямая продажа продукта | 12-20 сторис |

**КРИТИЧНО: Это МАКСИМУМЫ. Всегда стремись к нижней границе. Лучше 12 огненных сторис, чем 25 размазанных. Если можешь сказать за 15 — не растягивай на 25.**

### Фаза 3: Создание линии сторис

Для каждой сторис пиши в формате:

```
### Сторис [N]

**Тип:** [говорящая голова / текстовый слайд / инфографика / скриншот / интерактив]
**Визуал:** [описание того, что на экране]
**Текст на сторис:** [короткий текст-якорь на экране — MAX 10 слов. Для говорящей головы это подпись/тезис, НЕ весь контент]
**Голос:** [что говорить на камеру — это ОСНОВНОЙ контент для говорящей головы. Разговорно, живо, в ToV Яско. Пиши как реальную речь.]
**Интерактив:** [опрос/кнопка/стикер, если есть]
**Функция:** [зачем эта сторис в линии — 3-5 слов]
```

### Фаза 4: Проверка линии

Пройдись по чеклисту:
- [ ] Первая сторис — лёгкий Life-контент или цепляющий хук (НЕ «Привет»)?
- [ ] Тема понятна за 2-3 сторис?
- [ ] Есть интерактив в первых 5 сторис?
- [ ] Минимум 50% сторис — говорящая голова (основной контент идёт через речь)?
- [ ] Чередуются типы (голова → слайд → скриншот → интерактив)?
- [ ] Есть социальное доказательство (скриншоты, кейсы)?
- [ ] В конце — резюме + призыв к действию?
- [ ] Текст на каждой сторис читается за 3 секунды?
- [ ] Заголовки пунктов — КАПСОМ на чёрном фоне?
- [ ] Нет перегруженных сторис (макс 3 коротких предложения)?
- [ ] Общее количество НЕ превышает верхнюю границу для типа линии?
- [ ] Каждая сторис = 1 мысль (не больше)?
- [ ] Стиль = ToV Яско (разговорно, прямо, с энергией)?

## Ключевые правила стиля

### Структура (база Тимочко):
1. **Заголовки пунктов** — КАПСОМ на чёрном фоне
2. **Провокационные названия** — яркие метафоры
3. **Математический подход** — цели через формулы
4. **Диалог с аудиторией** — имитируй вопросы зрителя
5. **Контрасты** — таблицы «правильно vs неправильно»
6. **Скриншоты** — после каждого экспертного блока
7. **Интерактив** — каждые 5-7 сторис
8. **Лёгкость** — между тяжёлыми блоками Life или юмор

### Усиление (контент-маркетинг):
9. **Хук без delay** — никаких «Привет, сегодня хочу поговорить...»
10. **Contrast word** в хуке — «но/без/хотя/на самом деле»
11. **Speed to value** — тема ясна за 2-3 сторис
12. **Negative frames** — «Не делай X» мощнее, чем «Делай Y»
13. **Walk-away value** — зритель может СДЕЛАТЬ что-то после просмотра

### Tone of Voice Яско (душа):
14. **Разговорный** — как в кафе с другом
15. **Прямолинейность** — провалы = доверие
16. **Энергия** — короткие импульсные сообщения
17. **Антиперфекционизм** — «криво, но быстро»
18. **Фирменные маркеры** — движ, залетай, машина, на коленке, гиперценность, стакать
19. **Мат только для усиления** — не фон, а пик эмоции

## Адаптация под разные типы экспертов

**Маркетолог/запуски** → Цифры, формулы, кейсы. Структура «ошибки» или «шаги».

**Психолог/коуч** → Истории трансформации, эмоции, «было/стало». Структура «привычки» или «мифы».

**Фитнес/здоровье** → Визуал, до/после, конкретные программы. Структура «ошибки» или «принципы».

**Бизнес/финансы** → Цифры дохода, скриншоты. Структура «шаги» или «формула».

**Творческая ниша** → Портфолио, процесс создания. Структура «секреты» или «закулисье».

## Важные правила

- Язык сторис = язык пользователя (адаптируй ToV Яско под контекст эксперта)
- Не добавляй информацию, которой нет в контексте эксперта
- Если пользователь даёт транскрипт — переупакуй в формат сторис
- Текст и голос ДОПОЛНЯЮТ, не дублируют. Текст — «заголовок», голос — «статья»
- Каждая сторис = 1 мысль. Не помещается → раздели на 2
- **НЕ начинай с «Привет»** — Life или визуальный удар
- **НЕ заканчивай без CTA**
- **НЕ генерируй больше верхней границы** для выбранного типа линии
- Голос на камеру — как человек РЕАЛЬНО говорит: с паузами, «короче», энергией
