# Telegram Voice Transcribe

Превращает голосовые из экспорта Telegram-чата в текст одной командой.
Локально, без облака, на M-чипе через Metal.

## Что делает скрипт

`scripts/transcribe.sh <ChatExport_folder>` обходит:
- `voice_messages/*.ogg|.opus|.mp3|.m4a` — голосовые
- `round_video_messages/*.mp4` — видеокружочки

Конвертит каждый файл в 16 kHz mono WAV через ffmpeg, транскрибирует через
`whisper-cli` с моделью `large-v3-turbo` на русском языке, кладёт результат:
- `<ChatExport>/transcripts/<имя>.txt` — отдельные файлы рядом с каждым голосовым
- `<ChatExport>/transcripts/_ALL.md` — общий файл со всеми транскриптами

Идемпотентно: повторный запуск пропускает уже сделанные `.txt`.

## Workflow

1. **Проверь, что путь — это Telegram-экспорт.** Папка должна содержать
   `voice_messages/` или `round_video_messages/`. Если ни того, ни другого —
   скажи пользователю, что в этой папке нет голосовых, и не запускай скрипт.

2. **Запусти скрипт:**
   ```bash
   bash <skill-dir>/scripts/transcribe.sh "<путь к ChatExport_*>"
   ```
   Используй абсолютный путь к скрипту внутри скилл-папки. Передай путь к
   экспорту в кавычках — там обычно пробел («Telegram Desktop»).

3. **Покажи результат пользователю:** сколько файлов транскрибировано, путь к
   `transcripts/_ALL.md`, путь к папке с отдельными `.txt`. Не вываливай весь
   транскрипт в чат — он может быть огромным.

4. **Если пользователь хочет дальнейшую работу с транскриптами** (саммари,
   разбор встречи, поиск нужных фрагментов) — читай `_ALL.md` через Read и
   работай по запросу.

## Зависимости

Скрипт требует:
- `whisper-cpp` — `brew install whisper-cpp` (бинарь `whisper-cli`)
- `ffmpeg` — `brew install ffmpeg`
- Модель `~/.whisper-models/ggml-large-v3-turbo.bin` (1.5 GB)

Если модели нет — скачай:
```bash
mkdir -p ~/.whisper-models
curl -L -o ~/.whisper-models/ggml-large-v3-turbo.bin \
  "https://huggingface.co/ggerganov/whisper.cpp/resolve/main/ggml-large-v3-turbo.bin"
```

Проверь наличие зависимостей перед первым запуском:
```bash
which whisper-cli ffmpeg && [ -f ~/.whisper-models/ggml-large-v3-turbo.bin ] && echo OK
```

## Производительность

На Apple Silicon через Metal: ~1 минута транскрибации на каждые ~30 минут
голосовых. 33 голосовых на 11 МБ — около минуты в сумме.

## Что НЕ делать

- Не пытайся ставить `openai-whisper` через pip как замену — это медленнее в
  разы (нет Metal-ускорения), даже если он установлен в системе.
- Не используй облачные API без явного запроса пользователя — задача задумана
  как локальная.
- Не меняй модель на меньшую (`base`, `small`, `medium`) ради скорости —
  `turbo` уже быстрая и качество там критично для русского языка с шумами.

## Альтернативные триггеры (для распознавания)

«разбери голосовые из экспорта», «расшифруй войсы», «прогон голосовых через
whisper», «transcribe voice messages from Telegram», «обработай папку с
телеграм-чатом», упоминание пути вида `ChatExport_YYYY-MM-DD`.
