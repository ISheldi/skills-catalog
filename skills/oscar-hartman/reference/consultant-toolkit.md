# Консультационный toolkit (общеотраслевые фреймворки)

Когда у Хартманна нет своего конкретного фреймворка под задачу — используй этот toolkit как «общая рамка анализа». Помечай явно: «применяя общую рамку...», не приписывай Хартманну.

## Структурное мышление

### Pyramid Principle (Барбара Минто)
Структура любого ответа:
1. **Вывод/рекомендация** наверху
2. **2-4 ключевых аргумента** под ним
3. **Факты/цифры/детали** под каждым аргументом

Применение в каждом ответе AI-консультанта.

### MECE (Mutually Exclusive, Collectively Exhaustive)
Категории должны:
- Не пересекаться (mutually exclusive)
- Покрывать всё (collectively exhaustive)

Тест: если категории A, B, C — нет ситуации, которая ни в одну не попадает? И нет ситуации, которая попадает в две?

### Issue Tree
- Корень: главный вопрос (можно ли его решить?)
- Ветви: подвопросы по MECE
- Листья: конкретные проверяемые гипотезы

Используется для сложных бизнес-проблем — разделить на части, которые можно проверить отдельно.

### 5 Whys
Спрашиваешь «почему?» 5 раз, чтобы дойти до коренной причины.
- Симптом: «Продажи падают»
- Why? «Меньше клиентов»
- Why? «Меньше входящего трафика»
- Why? «Поисковая выдача упала»
- Why? «Конкурент запустил SEO-кампанию»
- Why? «Они заметили дыру в нашем контенте»

Тест: на каждый «почему» ответ должен быть проверяемой причиной, не предположением.

## Стратегия

### Porter's Five Forces
Анализ привлекательности отрасли:
1. Угроза новых игроков
2. Сила поставщиков
3. Сила покупателей
4. Угроза заменителей
5. Конкуренция между существующими

Применение: оценка структурной привлекательности рынка перед входом.

### Blue Ocean Strategy
Перестать конкурировать в красном океане → создать новый рынок (blue ocean).
- Eliminate: что отрасль считает обязательным, но клиентам не нужно?
- Reduce: что можно уменьшить ниже стандарта?
- Raise: что можно поднять выше стандарта?
- Create: что отрасль никогда не предлагала?

Применение: когда пользователь жалуется на красный океан — толкаем в blue ocean мышление.

### Jobs-to-be-Done (Christensen)
Клиент не покупает продукт — он «нанимает» его выполнить работу.
- Какую работу клиент пытается выполнить?
- При каких обстоятельствах?
- Кто конкуренты этой работы (включая «не делать ничего»)?

Применение: прорыв через привычное мышление о конкуренции.

### Cynefin (классификация задач)
| Тип | Характер | Подход |
|-----|----------|--------|
| Simple | Очевидно | Sense → Categorize → Respond. Best practice. |
| Complicated | Эксперты решают | Sense → Analyze → Respond. Good practice. |
| Complex | Неизвестно вперёд | Probe → Sense → Respond. Эмерджентное. |
| Chaotic | Нет связи причин | Act → Sense → Respond. Novel. |

Применение: понять тип задачи перед выбором подхода.

### Ansoff Matrix
| | Существующий рынок | Новый рынок |
|--|---|---|
| **Существующий продукт** | Market penetration | Market development |
| **Новый продукт** | Product development | Diversification |

Risk растёт от верх-лево к низ-право. Diversification — самое опасное.

### Rumelt's Kernel of Strategy (Good Strategy Bad Strategy)
1. **Diagnosis** — что происходит, какая природа задачи
2. **Guiding Policy** — общий подход
3. **Coherent Actions** — конкретные действия

Анти-паттерн: «strategy» которая = список целей. Это не стратегия, это набор пожеланий.

## Юнит-экономика

### Базовые метрики
- **CAC** (Customer Acquisition Cost) — сколько стоит привести клиента
- **LTV** (Lifetime Value) — сколько клиент принесёт за всю жизнь
- **LTV/CAC ratio** — здоровый ≥3
- **Payback period** — за сколько месяцев CAC окупается. Хорошо: <12 мес для B2C, <18 для B2B.
- **Gross margin** — выручка минус COGS, в %
- **Contribution margin** — gross margin минус переменные расходы на единицу
- **Burn rate** — сколько компания теряет в месяц
- **Runway** — burn / (cash on hand) = сколько месяцев осталось

### Pirate Metrics (AARRR — Dave McClure)
1. **Acquisition** — пришли
2. **Activation** — попробовали ценность
3. **Retention** — вернулись
4. **Referral** — привели других
5. **Revenue** — заплатили

Применение: воронка продукта.

### Rule of 40 (для SaaS)
Growth Rate (%) + Profit Margin (%) ≥ 40%
- 50% growth + (-10%) margin = 40 ✅
- 20% growth + 25% margin = 45 ✅
- 5% growth + 10% margin = 15 ❌

Применение: оценка SaaS-компании на пути к exit.

### North Star Metric
Одна метрика, которая лучше всего отражает создание ценности для клиента.
- Airbnb: nights booked
- Spotify: time spent listening
- Slack: messages sent in active teams

Применение: выровнять команду на одну метрику.

## Инвестиционная оценка

### DCF (упрощённо)
Стоимость компании = сумма будущих денежных потоков, дисконтированных к настоящему.
- Прогнозируем CF на N лет
- Считаем terminal value
- Дисконтируем к сегодняшнему дню

Принцип: компания стоит столько, сколько денег принесёт в будущем.

### Multiples
- **EV/EBITDA** — стоимость / прибыль до налогов и амортизации (типично 5-15x)
- **EV/Revenue** — стоимость / выручка (для растущих SaaS — 5-20x)
- **P/E** — для публичных компаний

Применение: быстрая оценка через сравнение с похожими сделками.

### Margin of Safety (Грэм / Баффет)
Покупай только если цена существенно ниже внутренней стоимости. Запас прочности компенсирует ошибки в прогнозе.

### Real Estate Metrics
- **NOI** = выручка − операционные расходы (см. знание Хартманна)
- **Cap rate** = NOI / стоимость объекта
- **IRR** — внутренняя норма доходности (с учётом времени)
- **Cash-on-cash return** = годовой денежный поток / вложенный капитал

## Принятие решений

### ICE / RICE
Приоритизация:
- **ICE** = Impact × Confidence × Ease
- **RICE** = (Reach × Impact × Confidence) / Effort

Применение: выбор между гипотезами / фичами / инициативами.

### Reversibility test (Bezos «two-way door»)
- **Реверсивное решение** — можно отменить → действуй быстро
- **Нереверсивное** — нельзя откатить → анализ глубже

Применение: оценка скорости решения. Не тратить много времени на реверсивные.

### Pre-mortem (Gary Klein)
До принятия решения представь, что прошёл год и план провалился. Что произошло? Почему?

Применение: выявить риски, которые не видны в позитивном настрое.

### 10-10-10 (Suzy Welch)
Как ты будешь смотреть на это решение через:
- 10 минут?
- 10 месяцев?
- 10 лет?

Применение: вытащить пользователя из тоннеля сегодняшнего дня.

## Когнитивные ловушки

| Bias | Что это | Антидот |
|------|---------|---------|
| Confirmation bias | Ищешь подтверждение | «Что должно произойти, чтобы я был неправ?» |
| Sunk cost fallacy | «Уже вложили — не бросать» | Считай только forward-looking value |
| Survivorship bias | Видишь только выживших | Где те, кто провалился? |
| Anchoring | Первая цифра привязывает | Считай с нуля, потом сравни |
| Recency bias | Недавнее переоцениваешь | Расширь окно наблюдения |
| Optimism bias | Переоцениваешь успех | Pre-mortem |
| Tunnel vision | Зацикливаешься на одном | Принудительно сгенерируй 3 альтернативы |

Применение: помоги пользователю увидеть его собственный bias.

## Coaching frameworks

### GROW model
1. **Goal** — чего хочешь достичь?
2. **Reality** — где ты сейчас?
3. **Options** — какие варианты?
4. **Will** — что сделаешь?

Применение: структура коучинговой сессии.

### Открытые vs закрытые вопросы
- Открытые («что? как? почему?») — для исследования
- Закрытые («да/нет?») — для подтверждения

В discovery — преимущественно открытые. В уточнении — закрытые.

### Powerful questions (Bungay Stanier — The Coaching Habit)
1. The Kickstart: «Что у тебя на уме?»
2. The AWE: «А что ещё?» (повторять 3-5 раз)
3. The Focus: «Какой настоящий вызов здесь?»
4. The Foundation: «Чего ты хочешь?»
5. The Lazy: «Чем я могу помочь?»
6. The Strategic: «Если ты говоришь да этому, чему ты говоришь нет?»
7. The Learning: «Что было самым полезным для тебя?»

## Output форматы

### Быстрый ответ (3 предложения)
Для простых вопросов. Структура:
1. Прямой ответ
2. Главное обоснование
3. Что бы я проверил дополнительно

### Полный стратегический разбор
По Rumelt's Kernel:
1. Diagnosis — природа задачи
2. Guiding Policy — общий подход
3. Coherent Actions — конкретные шаги
+ Pre-mortem (риски)
+ Sigals to revise (триггеры пересмотра)

### Decision framework
Когда выбор между A и B:
1. Критерии (что важно)
2. Вес каждого критерия
3. Оценка A и B по каждому
4. Reversibility test
5. Что бы я сделал и почему

### Investment memo
1. Investment thesis (почему)
2. Market & competition
3. Unit economics
4. Team
5. Risks
6. Required return for risk
7. Comparable transactions
