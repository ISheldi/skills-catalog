# Workflow — План и апрув (НОВЫЙ улучшенный v3.0)

После онбординга (`workflows/onboarding.md`) — ОДНИМ сообщением выдай пользователю ВСЁ для апрува. Не пакетируй частями, не «черновик-каркас» — сразу готовое.

---

## Что в одном сообщении

### 1. Пять вариантов хука
По выбранной ИСВП-формуле + 1-2 mix-варианта. Разные углы:
- Провокация
- Личная история
- Вопрос-зеркало
- Контрарианский совет
- Парадокс

### 2. Полная таблица слайдов — С КОНКРЕТНЫМ ТЕКСТОМ

Не «тип слайда» абстрактно, а **готовый текст**. Колонки:

| # | Тип | HEADLINE | SUB | BODY | FOOTER |
|---|---|---|---|---|---|
| 01 | Cover (Higgsfield) | хук-фраза | подзаголовок | — | @vladyasko |
| 02 | Context / Setup | определение | в чём суть | 1-2 строки | @vladyasko |
| 03 | Insight | главный тезис | пояснение | — | @vladyasko |
| ... | ... | ... | ... | ... | ... |
| NN | CTA | финал | вопрос аудитории | — | @vladyasko (центр) |

Применяй правило **«Слайд 01 = хайповый, Слайд 02 = приземлённый контекст»** из `knowledge/structure_principles.md`.

### 3. Какое фото / какие крео

- **Формат A:** для каждого фото-слайда (1, 5, 9) — конкретное имя из `Фото для каруселей/photos_index.json` с указанием `outfit_set` и почему оно подходит. **Варьируй outfit_set между слайдами** — нельзя 2-3 подряд из одного сета. **НЕ открывай JPG/PNG через Read** — читай `photos_index.json` и `.thumbs/`.
- **Формат B:** для cover — короткое ТЗ на Higgsfield (что генерируем: статуя / 3D-объект / визуальный плакат), какая преамбула из `prompts/higgsfield_*.md`.
- **Микс:** обе версии.

### 4. Полный текст подписи под пост (на чистовик)

По шаблону `templates/caption_template.md` — НЕ «черновик-каркас», а готовый текст.
