# Workflow — Безопасная итерация Higgsfield (НОВОЕ правило v3.0)

Цель: не палить токены и quota, не зацикливаться в плохих генерациях.

---

## Параметры по умолчанию

- Модель: `nano_banana_2`
- Resolution: `1k` (этого достаточно для Instagram)
- НЕ передавать `quality` — модель не поддерживает (см. `knowledge/known_bugs.md` баг 7)

## Цикл «генерация → анализ → решение»

### Итерация N

1. **Генерация:**
   ```
   generate_image(
     model="nano_banana_2",
     prompt="<преамбула из prompts/> + <ТЗ>",
     aspect_ratio="4:5",
     resolution="1k"
   )
   → job_display(job_id)
   ```

2. **Скачать и проверить размеры:**
   ```bash
   curl -o "/tmp/test_N.png" "{image_url}"
   sips -g pixelWidth -g pixelHeight "/tmp/test_N.png"
   # если > 2000 → sips -Z 2000 "/tmp/test_N.png"
   ```

3. **Анализ:**
   - `Read` PNG (теперь безопасно, ≤ 2000px)
   - Оцени сам: соответствует ТЗ / шрифт правильный / композиция / нет артефактов
   - Если плохо → переписать промпт (новая итерация)
   - Если хорошо → принять и идти дальше

## Лимит — 3 итерации

- **Максимум 3 ретрая подряд** на один слайд.
- Если 3 итерации подряд плохо — **зови пользователя**. Не пытайся 4-ю и 5-ю — токены утекают, а проблема скорее всего в ТЗ или ограничениях модели, а не в формулировке промпта.

## Когда звать пользователя

- 3 неудачных ретрая подряд
- Модель упорно не воспроизводит кириллицу правильно → предложи Soul 2.0 в приложении (Soul ID `<ВАШ_SOUL_ID>`)
- Лицо плохое после 2 ретраев с фото-референсом → тоже Soul 2.0 в приложении
