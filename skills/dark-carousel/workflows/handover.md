# Финальная сдача карусели — структура папки

> Пользователь должен сразу видеть утверждённую карусель, не копаясь в служебных файлах.

---

## Правило

В `готовые/<название карусели>/` остаются **только финальные файлы**. Всё остальное → подпапка `черновики/`.

## Что лежит В КОРНЕ (финал)

```
готовые/<название>/
├── 01_<slug>.png       ← cover
├── 02_<slug>.png       ← (или .mp4 если видео-слайд)
├── 03_<slug>.png
├── ...
├── NN_<slug>.png       ← последний (обычно CTA)
└── подпись_под_пост.md  ← готовый текст подписи
```

## Что лежит В `черновики/`

```
готовые/<название>/черновики/
├── *_raw.png                ← Higgsfield исходники без overlay
├── *_A.png, *_B.png, ...    ← отвергнутые cover-варианты
├── 02_video_raw.mp4         ← оригинальное видео без overlay
├── generate_*.py            ← Python-скрипты рендера
├── build_*.py
├── slides_data.json         ← контент-файл
├── html/                    ← все HTML-исходники для Playwright
├── stas_preview/            ← превью фото для выбора
├── _cover_work/             ← временные ресайз-копии для Higgsfield
└── *_test_*.png, *_temp_*   ← любые временные файлы
```

## Когда чистить

**В конце сдачи карусели** — после того как пользователь утвердил финал. До этого момента всё может лежать вперемешку (это нормально, идёт работа).

Алгоритм сдачи:
1. Перерендерить финальные слайды (после всех правок)
2. Создать `черновики/` подпапку
3. Переместить туда всё кроме `01_*.png … NN_*.png` и `подпись_под_пост.md`
4. Показать пользователю чистую сетку 12 файлов

## Команды (для bash)

```bash
cd "готовые/<название>"
mkdir -p черновики
# переместить всё кроме финала и подписи
mv *_raw.png *_A.png *_B.png *_C.png *_D.png \
   generate_*.py build_*.py slides_data.json \
   html stas_preview _cover_work \
   черновики/ 2>/dev/null
```

(Обязательно `2>/dev/null` — некоторые из перечисленных файлов могут не существовать в этом проекте, и без подавления mv падает.)

## Если пользователь захочет пересобрать слайд

Все исходники лежат в `черновики/` — Python-скрипты, JSON, HTML. Можно зайти, починить, перерендерить, заменить файл в корне. История не теряется.
