# Workflow — Онбординг новой карусели

Когда пользователь даёт запрос (текст, ссылку, идею) — НЕ начинай сразу. Сначала собери 4 вопроса через `AskUserQuestion`.

---

## Блок 1 — три вопроса одним сообщением

### Вопрос 1 — «Источник контента?» (header: «Источник»)
- Текст / тезисы (уже в запросе) — *Тип input A*
- YouTube / статья / сайт (URL) — *Тип input B, я прочитаю через WebFetch*
- Идея / тема (опишу сам по контексту) — *Claude сам пишет с нуля*
- Скриншот / фото — *Анализирую визуально*

### Вопрос 2 — «Формат дизайна?» (header: «Формат»)
- **A — Бренд-личность** — фото пользователя + типографика, фон cream-beige, шрифт Inter, 11-12 слайдов, фото на 1, 5, 9. Для постов про личный путь / экспертность / философию. (См. `knowledge/visual_dna_format_A.md`)
- **B — Гибрид Higgsfield + HTML** — 1 cover через Higgsfield, остальные HTML/Playwright, фон с клеточкой, оранжевая закладка, чёрные карточки. Для постов со скринами / схемами / списками. (См. `knowledge/visual_dna_format_B.md`)
- **Не уверен** — предложу на основе контента

### Вопрос 3 — «Объём карусели?» (header: «Объём»)
- 8 слайдов — короткий тезис
- 11-12 слайдов — стандарт (рекомендую)
- 16 слайдов — гайд / список (как 12 архетипов)
- 20+ — пошаговая инструкция

---

## Блок 2 — после ответов на блок 1

### Вопрос 4 — «Стиль хука для обложки?» (header: «Хук»)
- **И+С** — Интрига + Срочность (провокация, mystery)
- **В+П** — Выгода + Причастность (результат + для кого)
- **И+В** — Интрига + Выгода (любопытство + что получишь)
- **Придумай сам** (по контексту темы) — рекомендую если не уверены

См. полные формулы → `knowledge/hooks_formulas.md`

---

## После всех 4 ответов

→ Переходи к `workflows/plan_and_approve.md` (выдай ОДНИМ сообщением: хуки + таблицу слайдов с текстом + фото/крео + подпись на чистовик).
