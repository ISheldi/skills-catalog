# Workflow — Видео-фоны на слайдах (Live slides)

Часть дизайн-системы DARK-карусели. Позволяет делать «живые» слайды: контент тот же, но фоном вместо бежевой сетки идёт видео. Применяется к **готовой карусели формата A/B** — поверх уже отрендеренных слайдов.

См. также: `materials/video_bg/INDEX.md` (библиотека клипов), `scripts/process_video_bg.py`.

---

## Когда применять

- Решение **по контенту**, не по фиксированным слотам. Бери слайды, которые выигрывают от движения: тезис, поворот, механика, эмоциональные/«говорящие» моменты.
- **Слайд 01 (обложка) — НЕ трогать** (там свой Higgsfield/фото-визуал по жёстким правилам).
- Обычно **2-4 видео-слайда** на карусель из 10-14. Не делать видео-фон на каждом — теряется ритм и вес растёт.
- CTA-слайд (последний) — обычно оставлять статикой.

---

## Библиотека клипов

- Готовые клипы: `materials/video_bg/*.mp4` — уже 1080×1350, ≤15 сек, H.264, IG-совместимы.
- Сырьё (любой формат/соотношение): кладётся в `materials/video_bg/raw/`, прогоняется через `scripts/process_video_bg.py`.
- Превью для подбора: `materials/video_bg/.thumbs/*.jpg` (540px) — смотреть их, **не** читать сами mp4.
- Теги клипов: `materials/video_bg/video_bg_index.json` + человекочитаемый `INDEX.md`.

## Правило подбора (рандом с тегами)

1. Читай `video_bg_index.json`, фильтруй по `mood` / `scene`, которые подходят слайду.
2. Внутри подходящих — выбирай **рандомно**, но **без повторов** в одной карусели.
3. Варьируй сцены между видео-слайдами (не два «говорящих головы» подряд).
4. Если тёмное видео под тёмный текст не читается — это решает scrim (см. ниже), цвет видео вторичен.

## Шаг 1 (разово при пополнении): обработка сырья

```bash
# закинуть видео в materials/video_bg/raw/, затем:
VENV="/Users/vladyasko/.claude/skills/dark-carousel/.venv"
"$VENV/bin/python" scripts/process_video_bg.py
```

Скрипт: cover-кроп под 1080×1350 из любого соотношения, трим до 15 сек, H.264+AAC+faststart, thumbnail среднего кадра. После — **протегировать** новые клипы в `video_bg_index.json` (смотреть thumbnails, описать mood/scene/tags).

## Шаг 2: оверлей контента (прозрачный PNG)

Видео-слайд = `видео-фон` + `прозрачный оверлей с контентом`. Оверлей рендерится из той же вёрстки слайда, но в **видео-варианте**:

- `body { background:transparent }`, рендер с `omit_background=True`.
- Добавить `.scrim` (затемнение) — иначе текст не читается поверх видео.
- **Светлый текст** вместо тёмного: headline `#FFFFFF`, body `#EFEAE0`, kicker/swipe/em — оранжевый `#FF7A45`, counter/nick — белые. Везде `text-shadow:0 2-4px ... rgba(0,0,0,.5-.6)`.
- Тёмная карточка `.card` (#0A0A0A) — оставлять как есть, она и так контрастна; тень сделать чёрной.

Scrim-пресеты (подобрать под расположение текста на слайде):
- текст в верхней-средней зоне: `linear-gradient(rgba(10,10,10,.58) 0%, .46 38%, .46 62%, .9 100%)`
- текст + карточка снизу: `linear-gradient(.6 0%, .42 30%, .3 50%, .55 100%)`

Эталоны генераторов оверлея: `examples/video_overlay/example_overlay_std.py` (тип std — заголовок+body) и `example_overlay_card.py` (тип card — с тёмной карточкой).

Рендер оверлея:
```bash
"$VENV/bin/python" render_NN_overlay.py   # → NN_overlay.png (прозрачный)
```

## Шаг 3: композит видео + оверлей

```bash
ffmpeg -y -i "<clip>.mp4" -i "NN_overlay.png" \
  -filter_complex "[0:v][1:v]overlay=0:0:format=auto" \
  -c:v libx264 -profile:v high -pix_fmt yuv420p -crf 20 \
  -c:a aac -b:a 128k -movflags +faststart \
  "NN_<name>_VIDEO.mp4"
```

Длина итогового слайда = длина клипа (≤15 сек). Если клип короче, можно зациклить: добавить `-stream_loop` к входу видео и `-t <sec>`.

## Шаг 4: self-check

Снять контрольный кадр из середины и прочитать (1080×1350, в пределах лимита):
```bash
ffmpeg -y -i "NN_<name>_VIDEO.mp4" -ss <t/2> -vframes 1 "черновики/check_NN.png"
```
Проверить: текст читается, оранжевые акценты на месте, нумерация/ник/стрелка видны, важное в видео не срезано кропом.

## Шаг 5: сдача

- Видео-слайды кладутся в `готовые/<название>/` рядом с PNG: `NN_<name>_VIDEO.mp4`.
- Соответствующий статичный `NN_*.png` оставить (как запасной вариант) или убрать в `черновики/` — по договорённости.
- Скрипты оверлея, `*_overlay.png/html`, контрольные кадры → `черновики/`.
- При заливке в IG слайд берётся либо PNG, либо MP4 — у обоих один номер позиции.

---

## Жёсткие правила видео-слайдов

1. **01 обложка — никогда не видео.**
2. **Все слайды одного соотношения** — видео строго 1080×1350, иначе IG ломает карусель.
3. **Текст на видео — только светлый + scrim.** Тёмный текст по видео запрещён (нечитаемо).
4. **Длина ≤ 15 сек**, MP4 H.264 + AAC + faststart.
5. **Без повторов клипа** в одной карусели; варьировать сцены.
6. **Не больше ~1/3 слайдов** с видео-фоном — иначе теряется editorial-ритм.
