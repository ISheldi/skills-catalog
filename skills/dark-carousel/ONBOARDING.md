# 🚀 ONBOARDING — DARK Carousel

Скилл генерации Instagram-каруселей в DARK editorial-стиле.
**Создано by Stanislav Tsys · Instagram [@stastsis](https://instagram.com/stastsis)**

Этот гайд проведёт тебя от распакованного архива до первой карусели за ~10 минут.

---

## 1. Что делает скилл

Превращает сырьё (текст, ссылку на YouTube/статью, идею или скриншот) в готовую Instagram-карусель:
- сам задаёт 4 вопроса (источник / формат / объём / стиль хука),
- собирает хуки, структуру слайдов и подпись под пост,
- показывает план на твой апрув,
- рендерит финальные слайды 1080×1350 (PNG, опционально с анимацией/видео-фоном).

**Два формата дизайна:**
- **A — Бренд-личность**: твои фото + типографика, фон cream-beige. Для постов про личный путь/экспертизу.
- **B — Гибрид Higgsfield + HTML**: обложка через Higgsfield, остальное — HTML/Playwright. Для списков/схем/гайдов.

## 2. Что нужно установить

| Компонент | Зачем | Обязателен? |
|---|---|---|
| **Claude Code** (или десктоп-приложение Claude) | запуск самого скилла | да |
| **Higgsfield MCP** | генерация фото/обложек (`nano_banana_2`) | для форматов с генерацией |
| **Python 3.11+** | локальный рендер слайдов | для рендера PNG/MP4 |
| **Playwright** (`pip install playwright && playwright install chromium`) | HTML → PNG/MP4 | для формата B и анимаций |
| **ffmpeg** | склейка видео-фонов | только для «живых» слайдов |

> Если Higgsfield MCP не подключён — скилл всё равно работает: ты можешь давать свои изображения/обложки вручную, а текстовые слайды рендерятся локально через HTML.

### 2.1 🔌 Подключение Higgsfield (MCP)

Higgsfield — это сервис AI-генерации картинок и видео. Скилл общается с ним через **MCP-сервер** (Higgsfield подключается к Claude как внешний инструмент). Что скилл использует:
- модель **`nano_banana_2`** — генерация фото и типографики для обложек/слайдов;
- генерацию видео — для динамических фонов (опционально);
- **Soul 2.0** (точное лицо/персонаж) — это делается **в приложении Higgsfield**, не через MCP.

**Что нужно:**
1. **Аккаунт Higgsfield** с активной подпиской (генерация — платная, по кредитам).
2. **Ключ/токен доступа** к MCP — берётся в личном кабинете Higgsfield (раздел интеграций / API / MCP).

**Как подключить к Claude:**
- **Десктоп-приложение Claude:** Настройки → Connectors (Коннекторы) / MCP → «Добавить» → вставить адрес MCP-сервера Higgsfield и токен из кабинета.
- **Claude Code (терминал):**
  ```bash
  claude mcp add higgsfield <адрес-MCP-сервера-из-кабинета-Higgsfield>
  ```
  (точный адрес и способ авторизации — на странице интеграций Higgsfield; следуй их официальной инструкции по MCP).

**Проверка:** после подключения в новой сессии спроси Claude «какие инструменты Higgsfield доступны» — должны появиться функции генерации изображения/видео. Если их нет — токен не подхватился, перепроверь шаг.

> ⚠️ Точные адрес MCP-сервера и формат токена выдаёт сам Higgsfield в твоём кабинете — здесь они не вшиты намеренно (у каждого свой аккаунт/ключ). Не используй чужие ключи.
>
> 💡 Можно и без Higgsfield: давай скиллу свои фото/обложки (форматы с генерацией заменятся ручными изображениями), а текстовые слайды всё равно рендерятся локально через HTML+шрифты из коробки.

## 3. Установка скилла

1. Распакуй архив. Внутри — папка `dark-carousel/`.
2. Положи её туда, где Claude Code ищет скиллы:
   - **глобально:** `~/.claude/skills/dark-carousel/`
   - **или в проекте:** `<твой-проект>/.claude/skills/dark-carousel/`
3. Перезапусти Claude Code (или начни новую сессию), чтобы скилл подхватился.

Проверка: набери `/carousel` — скилл должен откликнуться.

## 4. ⚙️ Замена плейсхолдеров (ВАЖНО)

Скилл переносимый, поэтому личные пути/ник вынесены в плейсхолдеры. Замени их под себя.

### 4.1 Путь к скиллу → `/Users/vladyasko/.claude/skills/dark-carousel`

Шрифты в шаблоне и venv для видео ссылаются на абсолютный путь. Выполни в папке скилла **одну команду** (подставит реальный путь автоматически):

```bash
cd ~/.claude/skills/dark-carousel          # или твой путь установки
SKILL_DIR="$(pwd)"
grep -rl "/Users/vladyasko/.claude/skills/dark-carousel" . --include="*.html" --include="*.md" --include="*.py" \
  | xargs sed -i '' "s#/Users/vladyasko/.claude/skills/dark-carousel#${SKILL_DIR}#g"
```
> На Linux вместо `sed -i ''` используй `sed -i`.

### 4.2 Твой Instagram-ник → `@vladyasko`

Открой `config/settings.yml`, найди:
```yaml
brand:
  nickname: "@vladyasko"    # ← ЗАМЕНИ на свой ник
```
Поменяй на свой, например `@my_account`. Этот ник рендерится в левом нижнем углу каждого слайда.

> Хочешь заменить ник и в текстах правил/шаблонов сразу? Из папки скилла:
> ```bash
> grep -rl "@vladyasko" . | xargs sed -i '' "s/@vladyasko/@my_account/g"
> ```

### 4.3 (Опционально) Higgsfield Soul ID → `<ВАШ_SOUL_ID>`

Если будешь делать слайды с точным своим лицом через Higgsfield Soul 2.0 — впиши свой Soul ID вместо `<ВАШ_SOUL_ID>` в `knowledge/known_bugs.md` и `workflows/safe_higgsfield_iteration.md`. Если не пользуешься Soul — игнорируй.

## 5. Быстрый старт

В Claude Code напиши:
```
/carousel
```
или просто: «сделай карусель про X», «давай карусель», «новый пост».

Скилл спросит:
1. **Источник** — текст / URL / идея / скриншот
2. **Формат** — A (бренд-личность) или B (гибрид)
3. **Объём** — 8 / 12 / 16 / 20+ слайдов
4. **Стиль хука** — формулы ИСВП (интрига/срочность/выгода/причастность)

Потом покажет план (хуки + таблицу слайдов + подпись). После твоего «ок» — рендерит.

### ⚙️ Скрипты из коробки — готовая архитектура рендера

| Скрипт | Что делает | Как запустить |
|---|---|---|
| `scripts/process_video_bg.py` | Батч-обработка видео-фонов: любой исходник из `raw/` → cover-кроп 1080×1350, тримминг ≤15с, H.264+AAC+faststart (IG-готово) + thumbnail для тегирования | `<venv>/bin/python scripts/process_video_bg.py` |
| `examples/.../generate_slides.py` | Генерирует HTML всех анимированных слайдов одной карусели. Комбо эффектов: maskReveal заголовка + number-roll + Ken Burns + card fadeUp + per-row typewriter + drawLine underline | `python3 generate_slides.py` |
| `examples/.../render_animations.py` | Батч-рендер HTML → MP4 (Playwright + ffmpeg), **последовательно** чтобы не уронить сессию. 8.5с на слайд (вступление + цикл) | `python3 render_animations.py` (или `03 07` для отдельных) |
| `examples/.../record_video.py` | Запись одного HTML-слайда в MP4 через Playwright + ffmpeg (базовый кирпич, на нём строится batch) | `python3 record_video.py` |
| `examples/video_overlay/example_overlay_std.py` | Прозрачный текст-оверлей поверх видео-фона (обычный слайд: scrim + хук + body) | `python3 example_overlay_std.py` |
| `examples/video_overlay/example_overlay_card.py` | Прозрачный оверлей с карточкой поверх видео-фона (слайд с тёмной карточкой) | `python3 example_overlay_card.py` |

> **Логика рендера:** `record_video.py` — атомарная запись одного слайда. `render_animations.py` оборачивает её в безопасный последовательный батч. `generate_slides.py` сначала пишет HTML, потом его рендерят. `process_video_bg.py` готовит сырьё для «живых» слайдов, а `example_overlay_*.py` показывают, как класть текст поверх видео. Эти скрипты в `examples/` — рабочий референс, копируй и адаптируй под свою карусель.
