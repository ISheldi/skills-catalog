# Структурные принципы карусели

## Базовая формула

1. **1 главная идея** (тезис карусели)
2. **2-3 supporting идеи**
3. **Формула / парадокс / поворот** в середине
4. **CTA** в финале

## Объёмы

- 8 слайдов — короткий тезис
- 11-12 слайдов — стандарт (рекомендую)
- 16 слайдов — гайд / список (как 12 архетипов)
- 20+ — пошаговая инструкция

## Новое правило (v3.0): Слайд 01 ≠ Слайд 02

Не имеет смысла сразу после хука вываливать контент. Нужен мостик.

- **Слайд 01 — хайповый ХУК.**
  - Формат A: фото автора + интригующая фраза (Higgsfield generation либо HTML-overlay поверх фото).
  - Формат B: Higgsfield generation (статуя / 3D-объект / визуальный плакат) с провокацией.
  - Цель: остановить скролл.
- **Слайд 02 — ПРИЗЕМЛЁННЫЙ контекст «о чём пойдёт речь».**
  - 3D-объект / интро / definition / setup проблемы.
  - НЕ продолжение хука, а посадка читателя: «вот контекст, теперь идём в суть».
  - Цель: дать понять формат и пообещать структуру.

Только **со слайда 03** начинается основной контент.

## 🔴 ПРАВИЛО 21: Слайд 02 = ОТВЕТ крупно, НЕ ВОПРОС

На втором слайде **крупным шрифтом должна стоять суть карусели (ответ)**, а не вопрос «о чём этот пост» / «что такое X» / «про что речь».

### ❌ АНТИПАТТЕРН (так НЕ делать)

```
[крупно центр:]
О ЧЁМ
ЭТОТ ПОСТ                          ← вопрос огромным шрифтом

[мелко снизу:]
почему «я хочу быть уверен в тебе» — это
не про любовь, а про страх. разбираем,
чем доверие отличается от контроля.   ← ответ маленьким текстом
```

**Почему плохо:**
- Холодный читатель за 3 секунды видит «О ЧЁМ ЭТОТ ПОСТ?», думает «ну расскажи же» и листает дальше — мелкий ответ не успевает прочитать
- Самое ценное место слайда (центр, крупно) занято бесполезным вопросом-заглушкой
- Это слабая копия структуры лендингов «Что такое X? — текст ниже» которая там работает потому что у читателя есть время скроллить

### ✅ ПАТТЕРН (так НАДО делать)

```
[крупно центр или верх:]
«Я ХОЧУ БЫТЬ
УВЕРЕН В ТЕБЕ»
— ЭТО ПРО СТРАХ
А НЕ ПРО ЛЮБОВЬ           ← сразу СУТЬ крупно, ОТВЕТ на немой вопрос

[мелко снизу:]
↓ разбираем, чем доверие отличается от контроля   ← обещание структуры
```

Или ещё компактнее — выбрать одну главную идею:

```
ДОВЕРИЕ ≠ КОНТРОЛЬ        ← мощный thesis крупно

[ниже мельче:]
почему «я хочу быть уверен» — это про страх
↓ разбираем по полочкам
```

### Правила переписывания слайда 02

1. **Никогда не ставь kicker / headline типа** «О ЧЁМ ПОСТ», «ЧТО ТАКОЕ X», «ПРО ЧТО РЕЧЬ», «ЧТО ДАЛЬШЕ», «КОНТЕКСТ», «INTRO». Это **вопросы и категории**, не контент.
2. **Кикером ставь мощную фразу из контента**, например `ГЛАВНАЯ МЫСЛЬ`, `КОРОТКО`, `ТЕЗИС`, или сразу первое слово сути.
3. **Крупным шрифтом — суть в 4-8 словах**: главная idea карусели одной фразой. Это «ответ» на немой вопрос читателя «о чём вообще речь?»
4. **Мелким снизу — обещание структуры**: «разбираем по полочкам», «6 кейсов из бизнеса», «3 шага как сделать сегодня». Это сигнал «дальше будет полезно, листай».

### Хороший пример (карусель «Теория игр», v3.1)

❌ Было:
```
ЧТО ТАКОЕ
ТЕОРИЯ ИГР               ← заголовок-вопрос
математика решений когда другой тоже решает  ← мелкий ответ
```

✅ Должно быть:
```
ТЕОРИЯ ИГР =
МАТЕМАТИКА РЕШЕНИЙ
КОГДА ДРУГОЙ
ТОЖЕ РЕШАЕТ              ← суть крупно сразу

↓ 6 ключевых игр + кейсы из бизнеса   ← обещание структуры мелко
```

## Слайд-палитра (типы)

- **Cover** — хук
- **Context** — приземление / definition / setup
- **Insight / Idea** — главный тезис
- **Supporting** — поддерживающий аргумент / пример
- **Formula / Equation** — визуальная формула
- **Paradox / Twist** — поворот, контринтуитив
- **Case / Example** — пример из жизни
- **Step** — шаг инструкции (для гайдов)
- **Dark side / Anti-pattern** — что НЕ делать
- **CTA** — финал + призыв
