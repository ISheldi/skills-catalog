# Известные баги и фиксы (не повторять)

## Баг 1: счётчик "NN/16" обрезается справа в финальном MP4

**Причина:** `cameraShake` или любой `transform: translate()` на `.frame` смещает контент за `overflow: hidden` границу. На некоторых кадрах субпиксельный сдвиг + non-monospace шрифт = обрез цифры.

**Фикс:**
- Шрифт счётчика — `'JetBrains Mono', monospace` (стабильная ширина каждого символа)
- `font-weight: 700` (а не 800 как в первой версии)
- `padding-right: 8px` для буфера
- `z-index: 6` (над bookmark и header)
- **НЕ применять transform к `.frame`** — никаких cameraShake, тряски, общих сдвигов фрейма. Если нужна тряска — на конкретный header или элемент, не на frame.

---

## Баг 1b: двузначные номера #10 / #11 / #12 обрезаются по ширине

**Причина:** `.num-wrap { width: 130px }` — рассчитан на одну цифру. Двузначные числа шире, второй символ скрывается за `overflow: hidden`. Также при `line-height: 130px == font-size: 130px` ascenders шрифта Inter 900 вылезают сверху и обрезаются.

**Фикс:**
- `.num-wrap { width: 280px; height: 160px; padding-top: 10px; box-sizing: border-box; }` — запас на 2-3 символа + воздух сверху
- `.num-roll { line-height: 150px; height: 150px; }` (больше font-size 130px)
- `.num-roll span { line-height: 150px; height: 150px; }`
- Соответственно `rollNum` translateY: `−1650px` (= 11 × 150px), не `−1430px`

---

## Баг 2: Cyrillic "плывёт" между слайдами через nano_banana_2

**Причина:** `nano_banana_2` генерирует каждое изображение независимо, не помнит шрифт прошлого. При запросе на серию — каждый слайд получает свой grotesque sans-serif (Helvetica vs Druk vs Anton).

**Фикс:** в каждый промпт ДОСЛОВНО копировать шрифтовую инструкцию (см. `prompts/higgsfield_photo.md` или `prompts/higgsfield_typography.md`). Это не идеально, но даёт максимально близкие шрифты.

---

## Баг 3: nano_banana_2 не воспроизводит лицо точно

**Причина:** ограничение модели. Даже с `medias: [{value: photo_id, role: "image"}]` лицо приблизительное.

**Фикс:** для слайдов, где важно точное лицо — пользователь делает их сам в Higgsfield-приложении (Soul 2.0, Soul ID `<ВАШ_SOUL_ID>`). MCP `soul_2` НЕ работает, только в приложении.

---

## Баг 4: рубли в текстах (запрещено)

**Правило:** никаких `₽ / руб / рублей`. Только `$ / долларов / без валюты`. Применяется и к текстам слайдов, и к подписям под пост.

---

## Баг 5: чистовик до апрува

**Правило:** ВСЕ тесты — на низком качестве (`quality: "480p"`, `resolution: "1k"`). Чистовик `1k` запускается только после явного апрува пользователя на тестовых рендерах. Никогда не идти сразу в `2k` — для Instagram это избыточно и съедает время/quota.

---

## Баг 6: `media_ids` параметр устарел

**Правило:** используй `medias: [{"value": media_id, "role": "image"}]`, НЕ `media_ids: [...]` (старый параметр, не работает в nano_banana_2 MCP).

---

## Баг 7: `nano_banana_2` не поддерживает `quality`

**Правило:** параметр `quality` модель игнорирует/отвергает. Управление качеством — только через `resolution: "1k"`. Для теста используем тот же 1k.
