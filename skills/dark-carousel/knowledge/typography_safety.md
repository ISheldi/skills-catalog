# Типографика — безопасные значения для русского текста

## Line-height (межстрочное расстояние)

Для русского текста с буквами **Й, Ё** (а также Щ, Ц с нижними хвостами) — **минимальный line-height = 1.0**, желательно **1.05**.

### Почему

Меньше 1.0 (например `0.93` или `0.95`) даёт стилистический эффект «сжатости» текста, но:
- Точки над **Й** на следующей строке наезжают на верх буквы предыдущей строки
- Хвосты **Щ, Ц** заходят за следующую строку
- Это особенно заметно на крупных заголовках (>80px)

### Таблица минимальных значений

| Размер шрифта | line-height минимум | Рекомендуемо |
|---|---|---|
| 24-60px (body) | 1.2 | 1.3-1.4 |
| 60-90px (sub-headline) | 1.0 | 1.05 |
| 90-120px (headline) | 1.0 | 1.05-1.1 |
| 120-180px (huge) | 1.0 | 1.05-1.1 |
| 180px+ (display) | 0.95 (допустимо если нет Й/Ё) | 1.0 |

### Правило
**Если в тексте есть Й или Ё** → line-height **минимум 1.0**, лучше 1.05.
Никогда не ставь меньше 1.0 в заголовках которые могут содержать Й/Ё.

### 🔴 Implementation checklist (правило, которое я регулярно нарушаю)

Перед каждым рендером проверять **в самом CSS-шаблоне**:

- `slide_template.html` / `slide_format_A.html` — `--hero-line-height` или `.hero { line-height: ... }` должен быть **≥ 1.05** (не 0.88, не 0.93, не 0.95).
- Любой custom Playwright-скрипт (`render_3d_*.py`, временные одноразовые рендеры) — тоже **≥ 1.05** в инлайн-стиле hero.
- Это касается даже когда «визуально хочется плотнее» — для русских заголовков с Й/Ё/Щ/Ц жертвовать читаемостью нельзя.

**Сигнал бага:** буквы Й/Ё/Ц в одной строке упираются в верх букв нижней строки → значит line-height < 1.0.

**Чек-команда перед рендером:**
```bash
grep -E "line-height|hero-line-height" slide_template.html *.py
```
Все значения должны быть ≥ 1.0.

## Letter-spacing (межбуквенное)

| Контекст | Значение |
|---|---|
| Крупные заголовки 80px+, weight 900 | `-2px` до `-3px` (плотно) |
| Средние заголовки 40-60px | `-1px` до `0` |
| Body 20-30px | `0` (нейтрально) |
| Kicker / монопространство | `+1px` до `+2px` (расширенно для эффекта) |

## Минимальные font-size — НЕ УМЕНЬШАТЬ НИЖЕ

Это абсолютные минимумы для **читабельности на смартфоне** в IG. Меньше = текст не читается.

| Тип текста на слайде | Минимум | Норма | Пример |
|---|---|---|---|
| Headline (главный заголовок) | 60px | 90-130px | «ДИЛЕММА ЗАКЛЮЧЁННОГО» |
| Sub-headline | 32px | 40-60px | «(стратегия зеркала)» |
| Body (основной текст, объяснения) | 22px | 26-32px | «Каждый выбирает сдать…» |
| Setup / описание правил | 20px | 22-26px | «Двое попались на афере…» |
| **Footer / action / итоговая мысль** | **20px** | **22-28px** | **«В жизни → тестируй…»** |
| Kicker (категория сверху) | 18px | 20-22px | «ИГРА 1» |
| Подпись к цифрам / комментарий | 18px | 20-22px | «война: оба раненые» |
| Footer-line (правый низ, мета) | 16px | 18px | «12 слайдов · разбор» |

**Самое нарушаемое мной — action и footer внизу слайда.** В попытке впихнуть всё в один слайд я их давлю до 14-16px. **Это запрещено.**

## 🔴 ПРАВИЛО: «Добавь слайд вместо сжатия шрифта»

> Если контент не помещается на слайд с **минимальными шрифтами** — это **сигнал что концепт нужно разбить на 2 слайда**, а не уменьшать текст.

### Алгоритм когда контент не лезет

1. **Сначала** — сократить формулировку (часто помогает: «Большую часть времени — голубь. Но с готовностью стать ястребом если нарвался на агрессора» → «Тактика: голубь по умолчанию, ястреб при нужде»)
2. **Если короче не сделать** — разбить на 2 слайда:
   - Слайд X: setup + payoffs + кейс
   - Слайд X+1: «Что делать» — крупно, отдельно
3. **НИКОГДА** — не уменьшать шрифт ниже минимума из таблицы

### 12 слайдов = ориентир, не догма

Когда пользователь говорит «11-12 слайдов» — это **ориентир, не жёсткий лимит**. Лимит Instagram — 20 слайдов.

**Лучше отдать 14 читабельных слайдов**, чем 12 где половина мелким текстом. На этапе планирования (`workflows/plan_and_approve.md`) предупреждать пользователя если контент не влезает в желаемое количество.

### Признаки что нужно дробить

- На слайде уже есть headline + setup + matrix/data + объяснение + кейс + action — это 6 блоков, точно разделить
- Action / итоговая мысль вынуждена быть мелче 18px
- Несколько кейсов на одном слайде (каждый кейс заслуживает своего слайда)

## Жирность (font-weight)

| Элемент | Weight |
|---|---|
| Headline / display | **900 (Black)** |
| Sub-headline / accent | **800** |
| Body | **600-700** |
| Sub-body / hint | **500** |
| Quiet / footer | **400-500** |

---

## Шрифты

- **Inter** — основа всех заголовков и body (900 / 800 / 700 / 600 / 500)
- **JetBrains Mono** — цифры, моноширинные элементы, kicker'ы, технические подписи (700 / 500)

**Запрет:** serif, italic, decorative, thin/light (weight <400)

---

## Контрастность

- Тёмный текст: `#1A1A1A` на cream `#EFEAE0` — достаточно контраста
- Светлый текст: `#F5F0E0` на dark `#0A0A0A` — достаточно контраста
- Оранжевый акцент: `#E85A2C` — только для kicker'ов, чисел, акцентных слов внутри текста (не как основной body)
- Серый footer: `#5A5A5A` на cream — допустимый минимум для второстепенного текста
