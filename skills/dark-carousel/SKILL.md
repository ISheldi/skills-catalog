## ⚠️ ЗАЩИТА ОТ ПАДЕНИЯ СЕССИИ (КРИТИЧНО)

API лимит **2000px на сторону** для запросов с многими изображениями. Если в чат попадает несколько PNG > 2000px — сессия падает с ошибкой `An image in the conversation exceeds the dimension limit for many-image requests (2000px)`.

Правила:
1. **НЕ читай > 5 PNG подряд** за один ответ.
2. **Перед `Read` PNG > 2000px** → `sips -Z 2000 file.png` (in-place).
3. Готовые слайды 1080×1350 — в пределах лимита, безопасно.
4. Higgsfield 1k ≈ 1024px — безопасно.
5. Фото-референсы пользователя — проверять `sips -g pixelWidth -g pixelHeight file.jpg`, если > 2000 → `sips -Z 2000`.
6. Batch (генерация / рендер) — **последовательно**, не параллельно.
7. При работе с 12+ слайдами — НЕ читать массив PNG в чат, генерировать локально, показывать 1-2 ключевых.

## ЖЁСТКИЕ ПРАВИЛА (НЕ НАРУШАТЬ)

1. **Слайд 1 = всегда фото / визуал** (формат A — фото автора; формат B — Higgsfield generation).
2. **Слайд 02 = ПРИЗЕМЛЁННЫЙ контекст** (мостик: definition / setup / 3D-объект). Не сразу контент после хука.
3. **Фото каждые 3-4 слайда** в формате A.
4. **Никакого текста в правом верхнем / правом нижнем углу** (запретные зоны IG).
5. **Ник `@vladyasko` — нижний левый** на каждом слайде (кроме финального CTA где он по центру крупно).
6. **Нумерация `NN/TT`** — верхний левый, шрифт `JetBrains Mono`.
7. **Тесты — `resolution: "1k"`** (`quality` не передавать, модель не поддерживает). Чистовик `1k` только после апрува.
8. **Хэштеги в подписи — НЕ использовать.**
9. **Первое слово подписи — НЕ «Я».**
10. **Максимум 2 цвета текста** на одном слайде.
11. **Никаких рублей** — только `$` или без валюты.
12. **Не больше 5 PNG в `Read` подряд.**
13. **Не применять transform к `.frame`** в анимациях (баг с обрезанием).
14. **Перед `Read` PNG > 2000px** — `sips -Z 2000` обязателен.
15. **Перед рендером — обязательное согласование** структуры через `workflows/plan_and_approve.md`.
16. **Higgsfield итерации — лимит 3 ретрая** на слайд, потом зови пользователя (см. `workflows/safe_higgsfield_iteration.md`).
17. **🔴 Принцип «холодного читателя»** — пиши контент для человека, который НЕ знает темы карусели до того как начал листать. Каждый слайд самодостаточен, все цифры/термины подписаны, все action имеют мостик. См. `knowledge/cold_reader_principle.md`. **Это самое нарушаемое мной правило — проверять на каждой карусели.**
18. **Line-height минимум 1.0** для крупных заголовков с русским текстом (Й/Ё/Щ/Ц наезжают на 0.93-0.95). См. `knowledge/typography_safety.md`.
19. **🗂️ Финальная сдача = только финал в корне.** В `готовые/<название>/` остаются ТОЛЬКО `01_*.png … NN_*.png` + `подпись_под_пост.md`. Всё остальное (Higgsfield raw-исходники `*_raw.png`, отвергнутые варианты `*_A/B/C.png`, Python-скрипты, JSON, `html/`, временные превью) → в подпапку `черновики/`. Пользователь должен сразу видеть утверждённую карусель, не копаясь в служебных файлах. См. `workflows/handover.md`.
20. **🔴 Минимальные шрифты — не сжимать, лучше +1 слайд.** Footer/action/итоговая мысль на слайде = минимум **20px**, body = минимум **22px**. Если контент не лезет — сокращай формулировку или **дроби концепт на 2 слайда**. 12 слайдов — это ориентир, не догма (лимит IG = 20). Лучше 14 читабельных, чем 12 мелких. См. `knowledge/typography_safety.md`.
21. **🔴 Слайд 02 = ОТВЕТ крупно, НЕ ВОПРОС.** Запрещены kicker/headline типа «О ЧЁМ ЭТОТ ПОСТ», «ЧТО ТАКОЕ X», «КОНТЕКСТ», «INTRO». Крупным шрифтом — сразу суть карусели одной фразой (то что обычно пишут мелким текстом снизу). Мелким снизу — обещание структуры («↓ 6 кейсов / разбираем по полочкам»). Не повторять название карусели крупным шрифтом. См. `knowledge/structure_principles.md` правило 21.
22. **🎬 Видео-фоны (живые слайды).** Можно делать слайды с видео-фоном: контент тот же, фоном вместо сетки идёт клип из библиотеки `materials/video_bg/`. Решение **по контенту** (2-4 слайда на карусель, НЕ обложка 01, обычно НЕ последний CTA). Подбор клипа — **рандом по тегам без повторов** в одной карусели. На видео текст **только светлый + scrim** (тёмный текст по видео нечитаем). Видео строго **1080×1350, ≤15 сек, H.264** (все слайды одного соотношения, иначе IG ломает карусель). Полный процесс — `workflows/video_backgrounds.md`.
