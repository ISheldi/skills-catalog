# Viral Threads Post Generator

Скилл для создания вирусных текстовых постов для Instagram Threads на основе обратного инжиниринга топ-креаторов (@heythemnaji — 58K+, @creatortadeaas — 19K+, @lifeasminee) и актуальных данных алгоритма 2026 года.

---

## КЛЮЧЕВЫЕ ПРИНЦИПЫ THREADS (2026)

### Как работает алгоритм
- **Реплаи > Посты**: Адам Моссери (глава Threads): «Если хотите расти — отвечайте больше, чем постите». Replies — это 50% просмотров на платформе.
- **Глубина > Лайки**: Тред с 50 репостами и 20 развёрнутыми комментами получит больше охвата, чем пост с 200 лайками без комментов.
- **Пик жизни поста**: 6-12 часов. Лучшее время: 8:00-10:00 утро и 18:00-21:00 вечер (по ЦА).
- **141.5 млн DAU** (январь 2026), медианный engagement rate 6.25% (у X — 3.6%).
- **Conversation-first**: Threads награждает тех, кто разговаривает, а не транслирует.
- **Частота**: минимум 1-3 поста в день + активные реплаи на чужие треды.
- **Лимит**: 500 символов на пост. Оптимально — до 300 символов.

### Тон и стиль (из анализа топ-аккаунтов)
- Пиши как в переписке с другом, НЕ как пресс-релиз
- Короткие строки. Каждое предложение — с новой строки
- Одна мысль = один пост
- Больше «How I» чем «How to» (личный опыт > абстрактные советы)
- Уязвимость побеждает. Открытость = доверие
- Никакого AI-slop. Человечность — главное конкурентное преимущество на Threads

2. **Ниша** — AI, маркетинг, бизнес, лайфстайл, продажи и т.д.

4. **Формат** — одиночный пост / мульти-тред (серия реплаев) / карусель

Если контекста достаточно — НЕ спрашивай, сразу генерируй. Действуй по принципу «результат → уточнение потом».

## 7 ВИРУСНЫХ ФОРМАТОВ ПОСТОВ (обратный инжиниринг)

### ФОРМАТ 1: 🏆 ПОБЕДЫ / WINS
**Почему работает**: Люди на Threads обожают праздновать чужие победы. Вызывает эмоции: вдохновение, мотивация, связь.
**Структура**:
```
[Крупная победа / достижение — на первых строках]
[Контекст: что это значит для тебя]
[Короткая предыстория (1-2 строки)]
[Эмоциональный аккорд]
```
**Примеры хуков**:
- «МЫ СДЕЛАЛИ ЭТО!!! Первые $1,000 онлайн!»
- «Только что проснулся и увидел 10,000 подписчиков. Год назад было 52.»
- «Мой первый пост набрал 47 просмотров. Вчера пост набрал 188K views.»

**Шаблон**:
```
[ЭМОЦИОНАЛЬНЫЙ ТРИГГЕР — победа крупными буквами]

[Что было раньше — 1 строка]
[Что стало сейчас — 1 строка]

[Благодарность/эмоция]
```

---

### ФОРМАТ 2: 🔄 ТРАНСФОРМАЦИЯ / BEFORE-PATH-AFTER
**Почему работает**: Люди запрограммированы на истории изменений. Контраст «тогда → сейчас» — самый мощный нарративный приём.
**Структура**:
```
[ДО: список негативов (3-5 пунктов)]
[СЕЙЧАС: список позитивов (3-5 пунктов)]
[Путь: как перешёл от одного к другому (1-3 строки)]
```
**Примеры хуков**:
- «Мне {возраст}. В {прошлый_возраст} я чувствовал {нег1, нег2, нег3}. Сейчас — {поз1, поз2, поз3}. Вот что я сделал:»
- «Я в 17: • Алкоголь • Тусовки • Видеоигры. Я в 20: • Спортзал • Учёба • Свой бизнес»
- «2 года назад: 52 подписчика. Сегодня: 58,000+. Вот 6 шаблонов, которые я использовал:»

**Шаблон**:
```
[Я/Мне] в [прошлом]:
• [негатив 1]
• [негатив 2]  
• [негатив 3]

[Я/Мне] сейчас:
• [позитив 1]
• [позитив 2]
• [позитив 3]

[1-2 строки: что изменило игру]
```

### ФОРМАТ 3: 📋 СПИСКИ / LISTICLE
**Почему работает**: Быстро читаются, предсказуемы (в хорошем смысле), высокий save-rate.
**Правило**: пункты пиши от коротких к длинным (визуальная лесенка).
**Структура**:
```
[Короткий хук — 1-2 строки]
[Нумерованный список 5-9 пунктов]
[Финальный takeaway — 1 строка]
```
**Примеры хуков**:
- «Нормализуй это к 25 годам:»
- «{X} уроков за {X} лет. Ты получишь их за 2 минуты:»
- «Вот что я бы сделал, если бы начинал с нуля в 2026:»

**Шаблон**:
```
[Заголовок-хук]:

1. [короткий пункт]
2. [средний пункт]
3. [средний пункт]
4. [длиннее]
5. [самый длинный и весомый]

[Финальная строка-вывод или CTA]
```

---

### ФОРМАТ 4: 🎯 ВЫЗОВ АУДИТОРИИ / CALL-OUT
**Почему работает**: Адресное обращение к конкретной группе создаёт ощущение «это про меня».
**Группировки для таргетинга**: по возрасту (20s, 30s, 40s), по идентичности (мужчины, женщины, родители), по роли (фаундеры, креаторы, корпораты, фрилансеры).
**Примеры хуков**:
- «Мне 30. Если тебе 20+ — прочитай это.»
- «Я {профессия}. Вот 5 уроков, которые я хотел бы знать раньше.»
- «Я {профессия}, и мне тяжело это говорить, но {контринтуитивный вывод}.»
- «Если ты работаешь в найме — прочитай:»

**Шаблон**:
```
[Кто ты + обращение к группе]

[Список инсайтов или один ключевой вывод]

[Почему это важно именно для ЭТОЙ аудитории]
```

### ФОРМАТ 5: 📰 ГОРЯЧИЕ НОВОСТИ / HOT NEWS
**Почему работает**: Ты заимствуешь вирусность самого события. Скорость реакции = охват.
**Структура**:
```
[Громкий факт / новость — первые 2 строки]
[Контекст: почему это важно]
[Твоя уникальная интерпретация / мнение]
```
**Примеры хуков**:
- «Threads обогнал X!!! 141.5 млн DAU vs 125 млн. Впервые в истории.»
- «OpenAI только что выпустил [X]. Вот что это значит для тебя:»
- «Meta запускает рекламу в Threads. Вот что изменится:»

**Шаблон**:
```
[НОВОСТЬ — 1-2 строки, крупно]

[Почему это важно — 1-2 строки]

[Твой take / что делать — 1-2 строки]
```

### ФОРМАТ 6: 🧵 VALUE THREAD (мульти-пост)
**Почему работает**: Фаворит @heythemnaji. Удерживает на профиле дольше → алгоритм считает контент качественным. Высокий save + repost.
**Структура**:
```
[Главный пост: хук 3-4 мощные строки + визуал]
  └→ [Реплай 1: пункт/урок]
  └→ [Реплай 2: пункт/урок]
  └→ [Реплай 3: пункт/урок]
  └→ ...
  └→ [Финальный реплай: CTA]
```
**Правила**:
- Хук в главном посте. Последняя строка хука = приглашение читать дальше
- 5-10 реплаев (каждый < 300 символов)
- Нумеруй реплаи (1/, 2/, 3/...)
- Финальный реплай: CTA + ссылка

**Примеры хуков**:
- «Я постил 500+ раз в Threads. Вот 7 типов постов, которые ВСЕГДА залетают (с реальными примерами):»
- «20 месяцев назад я пришёл в Threads с 52 подписчиками. Сегодня — 48,700+. Как? Вот 6 шаблонов:»
- «{X} лет опыта сжал в 2-минутное чтение. Вот {X} уроков:»

### ФОРМАТ 7: 🖼 ОДИНОЧНОЕ ИЗОБРАЖЕНИЕ / SINGLE IMAGE
**Почему работает**: Просто, быстро, останавливает скролл. Эстетичный визуал + одна строка хука = мгновенное внимание.
**Структура**:
```
[Однострочный хук — макс 1-2 предложения]
[Визуал: инфографика / цитата на тёмном фоне / скриншот / мем]
```
**Когда использовать**: для мемов, мотивационных цитат, скриншотов результатов, инфографик.

## Шаг 3: Генерация ХУКОВ (ОБЯЗАТЕЛЬНО 3 варианта)

Для КАЖДОГО поста генерируй РОВНО 3 варианта хуков разных типов.

### Библиотека типов хуков для Threads:

**ХУК A — CREDENTIALS + PROMISE (Доказательство + обещание)**
```
[Твоё достижение/опыт]. [Обещание что читатель получит].
```
Примеры:
- «Я изучал Threads 20 месяцев и набрал 48,000+. Вот 6 шаблонов (копируй и публикуй):»
- «147 проектов за 3 года. Вот формула, которая работала в каждом.»

**ХУК B — КОНТРАСТ / TRANSFORMATION**
```
[Негативная точка А] → [Позитивная точка Б]. [Как/Почему].
```
Примеры:
- «Мои посты набирали 47 просмотров. Потом я понял одну вещь — и следующий набрал 188K.»
- «Год назад — 0 подписчиков. Сегодня — 10,000+. Вот что я делал:»

**ХУК C — ПРОВОКАЦИЯ / BOLD CLAIM**
```
[Смелое/контринтуитивное заявление]. [Подкрепление].
```
Примеры:
- «Threads — самая лёгкая платформа для роста в 2026. Но 90% людей делают это неправильно.»
- «Прекрати винить алгоритм. Адаптируйся.»
- «Контент-план — худшее, что ты можешь сделать для своего Threads.»

**ХУК D — ВОПРОС-ТРИГГЕР**
```
[Вопрос, бьющий в боль ЦА]?
```
Примеры:
- «Постишь каждый день, а рост = 0? Вот в чём проблема:»
- «Почему твои треды набирают 50 просмотров, а у конкурента — 50K?»

**ХУК E — CURIOSITY GAP (Дыра любопытства)**
```
[Неожиданный факт / тизер]. [Пауза]. [Развитие].
```
Примеры:
- «Один пост не принёс лайков. Но принёс 50 подписчиков и 1 продажу. Вот урок:»
- «Есть один приём, о котором не говорят 95% экспертов по Threads.»

## ДОПОЛНИТЕЛЬНЫЕ РЕЖИМЫ

### Режим «Обратный инжиниринг»
Если пользователь даёт текст чужого поста или ссылку — разбери по формуле:
1. **Формат** — какой из 7 форматов использован
2. **Хук** — какой тип хука и почему он цепляет
3. **Структура** — какие блоки, в каком порядке
4. **CTA** — есть ли, какой тип
5. **Почему залетел** — конкретные причины успеха (эмоция, контраст, тайминг, тренд)
6. **Как адаптировать** — 3 варианта как переделать под тему пользователя

### Режим «Контент-план»
Если пользователь просит план — генерируй таблицу:

| День | Тема | Формат (из 7) | Тип хука | Цель | CTA |
|------|------|---------------|----------|------|-----|

**Правила плана:**
- Чередуй форматы (не повторяй один формат 2 дня подряд)
- 1-3 поста в день (основной + реплаи)
- 2-3 value-треда в неделю
- 1-2 hot news в неделю (если есть инфоповоды)
- Минимум 5-10 реплаев на чужие треды ежедневно (вынеси отдельной строкой)

### Режим «Адаптация контента»
Если пользователь даёт готовый текст (пост для другой платформы, статью, транскрипт) — сожми в формат Threads:
1. Выдели 1-3 ключевых инсайта
2. Для каждого инсайта: выбери формат из 7 + сгенерируй 3 хука
3. Собери готовые посты
4. Если контента много → предложи серию постов / value thread

### Режим «Серия постов из аккаунта»
Если пользователь даёт аккаунт-референс — проанализируй контент и:
1. Определи 3-5 повторяющихся форматов
2. Выдели топ-хуки (по структуре, не копируя)
3. Адаптируй под нишу и стиль пользователя
4. Сгенерируй 5-10 постов в стиле аккаунта, но с уникальным контентом пользователя

## ПРАВИЛА ГЕНЕРАЦИИ

1. **Язык** — живой, разговорный. Как сообщение другу. Без канцелярита и корпо-языка. Русский по умолчанию.
2. **Хуки** — ВСЕГДА 3 штуки, ВСЕГДА разных типов. Это нужно для тестирования.
3. **Длина поста** — до 300 символов для одиночных. До 500 символов макс. Короче = лучше.
4. **Строки** — каждое предложение с новой строки. Короткие предложения. Нет стен текста.
5. **Конкретика** — цифры, сроки, результаты. Не «много» → а «347».
6. **Без воды** — если строку можно убрать без потери смысла — убери.
7. **Эмоция > информация** — пост должен вызвать реакцию (удивление, несогласие, желание сохранить/переслать).
8. **Conversation starter** — каждый пост должен провоцировать дискуссию. Вопрос, мнение, или незавершённая мысль.
9. **Human first** — больше «я», «мне», «мой опыт». Меньше «как сделать X в 5 шагов». Threads любит личность, а не инструкции.
10. **Не звучи как AI** — никаких «В мире где...», «Давайте поговорим о...», «Многие не знают, но...» (только если иронично).
